# Asobi v0.2 ‚ÄĒ full de-MCP + make the graph answer the dispatcher's questions

**Date:** 2026-06-18 **Status:** IN_PROGRESS **Epic:** `asobi:v0.2` (canonical task state lives in the graph)

v0.2 finishes the de-MCP started in `chore/drop-mcp` and targets the weaknesses the daily skill workflows hammer: **write amplification**, **query power**, and **multi-agent reliability** ‚ÄĒ now with a clean, asobi-native CLI and db schema.

## Baseline (uncommitted on `chore/drop-mcp`, lands as its own commit)

- **Dropped the MCP stdio server** ‚ÄĒ `src/mcp.rs` deleted (‚ąí553); graph DTOs moved to `src/model.rs`; `mcp` command + `verbose_tool_test.rs` removed.
- **Removed the dead `ClaudeProvider` branch** (`ASOBI_EMBED_PROVIDER=claude`).
- **Batch writes** ‚ÄĒ `create-entities` repeated `NAME TYPE`; `create-relations` repeated `FROM TO TYPE`. _(Note: not yet in the installed 0.1.0 binary.)_
- **Global `--json`** ‚ÄĒ mutations echo affected entities to stdout.
- Docs de-MCP'd; new surface covered in `scripts/verify_cli.py`.

The `mcp_` db schema and the mcp-style command names were intentionally left in place here ‚ÄĒ v0.2 removes both (decisions `asobi:decision:schema-rename`, `asobi:decision:flat-verb-cli`, both extending `asobi:decision:drop-mcp`).

## Resolved decisions (2026-06-18)

1. **status-as-truth** ‚ÄĒ task/session current status is a **truth**, not an observation; observations are reserved for transition notes. Board renders from one `search --where status=‚Ä¶`. _(Adopted.)_
2. **CLI re-brand ‚Üí flat terse verbs**, **hard cut** (no deprecated aliases). `new ¬∑ obs ¬∑ link ¬∑ truth ¬∑ graph ¬∑ search ¬∑ show ¬∑ rm ¬∑ rm-obs ¬∑ unlink ¬∑ rm-truth`. Unchanged: `query ¬∑ ingest ¬∑ compact ¬∑ init ¬∑ backup ¬∑ restore ¬∑ skills ¬∑ stats`.
3. **Schema rename `mcp_` ‚Üí asobi-native**, **included now** (not deferred) ‚ÄĒ full de-MCP, with a migration for existing dbs + backup/restore changes.

### task-1 ¬∑ WAL + busy*timeout *(reliability, ~5 lines)\_

`PRAGMA journal_mode=WAL; synchronous=NORMAL; busy_timeout=<ms>` in `init_db()`. De-risks concurrent lead+sub-agent writes. **Done:** concurrent writers don't error on lock; `make check` green.

### task-2 ¬∑ Schema rename `mcp_` ‚Üí asobi-native _(correctness-critical)_

Rename `mcp_truths` + any `mcp_`-prefixed tables/SQL functions in `src/db.rs`; add a migration for existing 0.1.0 dbs (rename on open); update `src/backup.rs` dump+restore and `src/model.rs` if it names tables. **Done:** fresh db uses `asobi_` names; an existing db migrates cleanly; backup‚Üírestore round-trips.

### task-3 ¬∑ CLI re-brand ‚Üí flat terse verbs _(hard cut)_

Rewrite the clap command enum + dispatch in `src/main.rs`. Mapping: `create-entities‚Üínew ¬∑ add-observations‚Üíobs ¬∑ create-relations‚Üílink ¬∑ delete-entities‚Üírm ¬∑ delete-observations‚Üírm-obs ¬∑ delete-relations‚Üíunlink ¬∑ read-graph‚Üígraph ¬∑ search-nodes‚Üísearch ¬∑ open-nodes‚Üíshow ¬∑ add-truth‚Üítruth ¬∑ delete-truth‚Üírm-truth`. Then flip **every** consumer: `SKILL.md`, `docs/usage.md`, `docs/architecture.md`, `README.md`, `CLAUDE.md`, `scripts/verify_cli.py`, `tests/*`, and the `harus-skills:asobi` skill doc. **Done:** no old name resolves; `verify_cli.py` + `make check` green; grep finds zero old names in docs/scripts/tests.

### task-4 ¬∑ `query --json --limit N` _(knowledge-tier parity)_

`src/main.rs` Query arm + `src/recall.rs` hardcode `5` and a human `{:<20} | score | path` format. Add `--limit` (default 5) and `--json`.
